Pages

The project examples listed below that showcase my capabilities as lead front-end developer.

Adecco USA

adeccousa.com/employers/resources/living-wage-tool/

I took ownership of Adecco USA’s website in February 2021. At that time I was tasked with maintenance and development of existing systems to support the ongoing needs of the business. One new task was the creation of the Cost of Living Calculator. I worked with a business analyst, a web designer, a data engineer, and a fellow front-end developer to bring each aspect of the project to the web site. I used React to build the component that bring each individual effort together.

The Park School

parkschool.org/community

The client wanted to feature their logo as a way to engage users to delve deeper into the website. I disassembled an SVG of their logo, rebuilt it with HTML, and used CSS and JavaScript to make it interactive.

COBIS

cobis.org.uk/our-network/search-for-cobis-members/cobis-school-search

The client wanted a way to display their 250 schools. I utilized the Google Sheet API to populate the content on this page with the Google Map API.

Gill St. Bernard’s

gsbschool.org/academics

The client wanted to have unique ways for the users to interact with landing pages for each section of this site. This page utilizes various callout boxes the client used for an image, content, and a call to action button. The section at the bottom (Admission Process, Affording Gill, Campus Tour, FAQs, Request Information) uses a custom slideshow to cycle through those images. Once the user clicks on the plus sign to view more content there is a slideshow that allows the user to cycle through all of the items.

Rye Country Day School

ryecountryday.org

The client wanted to have custom treatments for each section of this homepage. Each section exemplifies a different design challenge that can stand on its own but works well in the overall functionality of the website.

Magen David Yeshivah School

magendavidyeshivah.org

The client makes use of a unique setup. The video loads for the user only the first time visiting the page. Once a user watches the entire video or chooses to skip it the rest of the page content is displayed by setting a cookie. The other elements on the page use a balance of CSS and JavaScript for the animation. The cookie is cleared once the user closes their browser.
